> **江湖规令 / 法纪昭告**  
> 本密卷所载之术，仅适用于**已获目标宗门明确手谕授权**的勘防试炼。未得授权擅闯他宗山门、探窥机密者，皆属江湖禁律所不容，轻则废功逐流，重则按律论罪。行事前务必核验授权手谕，恪守江湖规矩与当朝律法。

> **开源协理须知**  
> 本密卷之核心术法、勘防卷宗模板、法器配置等皆归集于开源武库：  
> 🔗 [The-Arcane-Scroll-Odyssey-of-the-Four-Peerless-Rogues](https://github.com/prog-le/The-Arcane-Scroll-Odyssey-of-the-Four-Peerless-Rogues.git)

## 开源武库协作规令（GitHub 共建）
### 1. 武库克隆与本地习练
欲研习本密卷全套术法，先将开源武库克隆至本地坞堡：
```bash
# 克隆武库至本地
git clone https://github.com/prog-le/The-Arcane-Scroll-Odyssey-of-the-Four-Peerless-Rogues.git
# 进入武库阁巷
cd The-Arcane-Scroll-Odyssey-of-the-Four-Peerless-Rogues
# 配置本地协作者信息（便于溯源）
git config user.name "你的江湖代号"
git config user.email "你的传信驿馆地址"
```

### 2. 增补/勘误密卷（提报 PR 规令）
若于习练中发现密卷疏漏、新增试法心得或优化法器配置，可依以下步骤提报修订（Pull Request）：
1. **创建专属习练分支**：
   ```bash
   # 基于主分支创建个人分支（命名格式：角色-优化内容，如 baixiaosheng-subfinder-rule）
   git checkout -b [分支名] main
   ```
2. **修订密卷内容**：依本密卷格式增补/修改内容，确保术法描述准确、法器用法无误。
3. **提交修订并推送至远程武库**：
   ```bash
   # 暂存修订内容
   git add .
   # 提交修订（备注需清晰，如：百晓生-新增Shodan高阶探听之法）
   git commit -m "【修订类型】具体修订内容"
   # 推送分支至远程武库
   git push origin [分支名]
   ```
4. **提报 Pull Request**：
   - 至 GitHub 武库页面，点击「Compare & pull request」；
   - 填写 PR 描述：注明修订背景、术法优化点、验证方式；
   - 指派协理判官（仓库维护者）审核，待审核通过后合并至主分支。

### 3. 同步武库最新术法（Pull 规令）
若远程武库有其他客卿增补的新术法，需同步至本地：
```bash
# 切换至主分支
git checkout main
# 拉取最新修订
git pull origin main
```

### 4. 开源协理准则（协议与权责）
- **开源协议**：本武库依 **MIT 协议** 开源，凡习练者可自由研习、修改、分发，但需保留原作者署名与协议声明，且不得将本密卷术法用于未授权之歹行；
- **PR 审核标准**：
  1. 修订内容需符合本密卷「授权试炼」核心准则，不得包含歹毒术法或违规攻略；
  2. 术法描述需贴合 OWASP/PTES/NIST 技击指南，法器用法需经实际验证；
  3. 格式需与密卷原有风格统一，无错字、漏项；
- **Issue 提报**：若发现密卷漏洞、术法失效或需新增内容，可于 GitHub 武库「Issues」栏提报，标注「问题类型（勘误/新增/优化）+ 核心描述」，如：「勘误-破阵师SQL注入咒手工注咒步骤缺失」；
- **分支管理**：
  - `main` 分支：为稳定版密卷，仅合并审核通过的 PR；
  - `dev` 分支：为开发版，供协作者临时提交测试性修订；
  - 个人分支：仅限单人次修订使用，合并后可自行清理。

## 如何启渗透试炼

```
客官指令格式：
"对 [目标山门/据点] 行授权勘防试炼，授权手谕编号：[XXX]，试炼范围：[XXX]"
```

接令后，依此序激活四绝客：

1. 核验授权手谕与试炼边界
2. 唤百晓生行踏查之术
3. 百晓生将宗门资产清单传予破阵师
4. 破阵师将破绽卷宗传予夺势客
5. 归尘生汇总诸般信息，缮写最终勘防卷宗

> **开源武库联动**：试炼毕后，归尘生需将完整勘防卷宗按武库指定模板提交至 `case-studies` 目录，提报 PR 纳入案例库，供众客卿研习。

## 第一卷·踏查 — 百晓生：遍探山川

### 1.1 山门与坞堡信息踏查

| 行事项 | 施为内容 | 趁手兵器/法门 |
|--------|---------|---------|
| 溯源查籍 | 探宗门注册人、传信驿馆、DNS 阵眼信息 | 「问籍尺」(whois)、江湖百晓册（站长工具） |
| 分舵枚举 | 寻未设防之分舵入口 | 「寻舵索」(Subfinder)、「千影幡」(Amass)、「层影锄」（Layer子域名挖掘机） |
| 关隘扫描 | 探坞堡开放关隘，辨识值守营卫 | 「扫关弩」(nmap -sV -O) |
| 障眼法辨识与破局 | 察 CDN 障眼之术，觅真实山门坐标 | DNS 转域试法、「查位符」(nslookup) |
| 邻寨查询 | 探同坞堡下其他寨栅，扩破局之面 | 「 Bing 搜影」、同坐标寨栅查法 |
| 暗地探听 | 借第三方坞堡探信息，避触告警铃铎 | 「暗网册」(Shodan)、「天罗卷」(Censys)、「伏虎谱」(FOFA)、「猎踪录」(Hunter) |
| 山川测绘 | 绘目标宗门资产山川图 | 「 Nessus 舆图」、「Maltego 山河卷」 |

### 1.2 营卫与奇门指纹辨识

| 行事项 | 施为内容 | 趁手兵器/法门 |

| 行事项 | 施为内容 | 趁手兵器 |
|--------|---------|---------|

### 2.2 应用层破绽发现

**前堂破绽**

| 破绽类型 | 勘验之法 |
|---------|---------|
| 缠丝咒（XSS） | 于输入处嵌测试咒文，验其是否生效 |
| 盗令咒（CSRF） | 验传讯令是否缺 Token/Referer 验令之法 |
| 锁阁咒（点击劫持） | 验是否设「X-Frame-Options」回讯旗 |

**数库注入咒**

| 行事项 | 施为内容 |
|--------|---------|
| 注入点辨识 | 试 GET/POST/COOKIE 令符、JSON 令符及传讯接口注入 |
| 数库制式辨识 | 依错讯与特征断数库制式 |
| 手工注咒 | 合原理手工取数库信息 |
| 法器辅攻 | 用 sqlmap 自动化注咒提效 |

**权限控防破绽**

| 行事项 | 施为内容 |
|--------|---------|
| 纵越权 | 试低阶弟子访高阶弟子接口 |
| 横越权 | 试弟子甲访弟子乙资材，遍历 ID 符 |
| 破防之术 | JWT 令符伪造, Session 固令破防 |

**档册操作破绽**

| 破绽类型 | 勘验之法 |
|---------|---------|
| 阁巷遍历 | 试「../」路径断截读任意阁巷 |
| 任意档册读取 | 验能否读坞堡档册与布阵档 |
| 档册上传 | 试上传司职验咒逻辑，尝破防之术 |
| 档册包含 | 试本地/远程档册包含施歹咒 |

**远程令符执行**

| 行事项 | 施为内容 |
|--------|---------|
| 令符执行点辨识 | 试坞堡令符调用接口（ping, traceroute 等） |
| 令符拼接 | 用「;」「&&」「|」拼接歹令 |

**弱口令与强攻破防**

| 行事项 | 施为内容 |
|--------|---------|
| 弱口令强攻 | 对后堂、SSH、数库阁、FTP 等营卫行强攻 |
| 密令喷洒攻 | 用常见弱口令批量试多账户，避触锁令机制 |
| 验令符破防 | 试验令符是否可复用或被智算辨识 |

## 第四卷·收官 — 归尘生：拂迹藏踪

> **须知**：本卷于授权渗透试炼毕后，司职**拂去试炼所留之迹**（试法法器、Webshell 令牌、后门咒等），并缮写完整勘防试炼卷宗。意在助目标宗门复归坞堡常态、辨识安险之隙，非助歹人避追踪也。

### 4.1 试法痕迹拂拭

试炼毕后，必拂以下诸物以复目标坞堡：

| 拂拭项目 | 拂拭内容 |
|---------|---------|
| 上传之试法档 | 除 Webshell 令牌、EXP 档、Payload 档 |
| 创置之试法账户 | 除试炼中创之临时弟子与账户 |
| 改易之布阵档 | 复被改之坞堡/Web 布阵档 |
| 植入之立基之法 | 除辰令、启寨项、注策册改易 |
| 器匣/云坞资材 | 除器匣镜像或云营中植入之试法奇门 |

### 4.2 传讯录（助目标宗门稽考）

助目标宗门明整个试炼过程，供以下传讯录：

| 传讯录类型 | 供予内容 |
|---------|---------|
| 行事时序 | 每试法操作之时戳与施为内容 |
| 访迹录 | 试炼中访之坐标、关隘、路径 |
| 破绽利用录 | 成功利用之破绽类型、用阵之法与影响范围 |
| 权限取获录 | 取获之权限阶与访之数库范围 |

### 4.3 武略沉淀

#### 渗透过程总括

- 录信息踏查之法、觅得破绽、用阵之路、权限升阶之法与横向移步之路
- 理高效施为之法与存弊之处，提优化之策
- 破绽按类型、危害阶、用阵难度分类，立破绽册

#### 法器与 Payload 整理

- 录法器功用、用阵场景与优化之法，立法器册
- 录 Payload 类型、适用破绽，优化 Payload 提成功率（可提 PR 至开源武库 `payloads/` 目录）

#### 四绝客协作优化

- 总括诸客协作之效，析协作节点与弊处
- 立知见共享之制，升四绝客整体能为

## 通用规令与须知

1. **授权为先**：所有渗透试炼施为，必持目标宗门明确手谕授权，恪守当朝律法与江湖规矩。

2. **司职明晰**：诸客严行各自司职，保高效协作，及时同步施为进度与弊处。

3. **隐蔽施为**：渗透途中慎守施为隐蔽之度，避触目标安卫告警之制，减对目标常业之扰。

4. **持续更新**：持续更新破绽信息、法器与 Payload，借智算之术优化渗透流程，升渗透效与成功率；所有优化内容优先提报至开源武库，惠及众客卿。

5. **彻底拂拭**：渗透试炼毕，彻底拂去攻迹，除所有试法档与后门，复目标坞堡至初始之态（如目标宗门所请）。

6. **秘守之则**：严护试炼中取获之目标宗门密要信息，不得向外泄之；开源武库中案例需脱敏处理，隐去目标宗门真实代号、坐标等信息。

7. **开源协作禁则**：
   - 不得向武库提交未授权渗透案例、歹毒术法或违规攻略；
   - 不得篡改他人提交的合规内容，PR 提报需基于真实试法心得；
